0개요 & 준비물

여러 공간의 환경변수(온도·습도·기압·가스 등)를 무선으로 모아 한 화면에서 보는 시스템입니다. 센서를 바꾸면 다른 환경변수로도 확장됩니다.

무엇을 만드나

각 공간(예: 교실)에 센서 노드(Nano ESP32 + BME688)를 두고, 5분마다 측정값을 WiFi로 허브(UNO Q)에 보냅니다. 허브는 데이터를 데이터베이스에 쌓고, 웹 대시보드로 보여줍니다. 노드를 늘리면 화면에 자동으로 칸이 추가됩니다.

1허브 — 작업 시작 (WiFi + 작업 방식)

제일 먼저 허브(UNO Q)를 준비합니다. 허브를 WiFi에 연결하고, 작업을 어디서 할지(직접/SSH) 정합니다. 이어서 브로커(패널 2)·hub.py(패널 3)까지 마치면 허브가 완성되고, 그때 여기서 정한 브로커 주소를 들고 노드(패널 4~6)로 넘어갑니다.

1) 허브를 WiFi에 연결 — nmtui (가장 직관적)

UNO Q를 노드·대시보드와 같은 WiFi에 연결합니다. nmtui는 메뉴식 텍스트 GUI라 명령을 외울 필요가 없습니다.

sudo nmtui
  • Activate a connection → 목록에서 현장 WiFi(SSID) 선택 → 암호 입력 → 연결
  • 방향키 이동, Enter 선택, Esc 뒤로. 한 번 연결하면 부팅 시 자동 재연결됩니다(무인 운영의 전제).
▸ 명령(nmcli)으로 하거나 막힐 때 — 펼쳐보기

nmtui가 안 되거나 원인 진단이 필요할 때 쓰는 방법입니다. 알아두면 유용합니다.

① 보드가 보는 WiFi 목록 확인

nmcli radio wifi              # enabled 여야 함 (disabled면 sudo nmcli radio wifi on)
nmcli device wifi list        # 보드가 잡는 네트워크 목록
BSSID              SSID            MODE   CHAN  RATE       SIGNAL  SECURITY
06:29:D5:24:7E:71  wi_cne_class_T  Infra  11    260 Mbit/s  70      WPA1 WPA2
                                          ← 1~13=2.4GHz / 36↑=5GHz    ← 신호(높을수록 좋음)
목록으로 원인 판별찾는 SSID 보임 → ②로. 다른 건 뜨는데 그것만 없음 → 5GHz 전용 가능성(노드는 2.4GHz 필요). 목록이 빔 → WiFi 비활성/드라이버 확인.

② 연결 (가장 흔한 정답)

sudo nmcli device wifi connect "SSID이름" password "와이파이암호"

성공 시 Device 'wlan0' successfully activated....

⚠️ hidden yes를 함부로 붙이지 말 것진짜 숨김(SSID broadcast 끔) 네트워크에서만 씁니다. 일반 네트워크에 붙이면 오히려 No network with SSID '...' found 오류. wifi list에 보이면 숨김이 아니므로 빼고 연결하세요.

③ 숨김 / 같은 이름 AP 여러 대

# 진짜 숨김일 때만
sudo nmcli device wifi connect "SSID이름" password "암호" hidden yes
# 같은 이름 AP 여럿(중계기 등) — BSSID 직접 지정
sudo nmcli device wifi connect "06:29:D5:24:7E:71" password "암호"

2) IP 확인 — 노드 BROKER에 쓸 주소

hostname -I                        # 받은 IP (192.168.x.x). 노드 BROKER에 사용
nmcli connection show --active     # 연결된 SSID 확인
✅ 여기서 얻은 IP를 노드에hostname -I의 192.168.x.x → 노드 펌웨어 BROKER="192.168.x.x". DHCP라 재부팅 시 IP가 바뀔 수 있으니, 라우터 DHCP 예약으로 고정하면 노드를 다시 안 고쳐도 됩니다(강력 권장).
💡 노드(ESP32)는 2.4GHz만노드로 쓸 Nano ESP32는 2.4GHz 전용입니다. 허브·노드를 모두 CHAN 1~13(2.4GHz) SSID에 붙이세요. 라우터가 2.4/5GHz를 같은 이름으로 쓰면 2.4GHz 전용 SSID를 두거나 폰 핫스팟 2.4GHz 모드로 테스트.

3) 이제부터 허브 작업을 어디서 할까? (선택)

다음 단계(브로커·hub.py·대시보드·systemd)는 모두 허브에서 명령을 입력하는 작업입니다. 두 가지 방법이 있고, 입력하는 명령은 똑같습니다 — 접속 통로만 다릅니다.

UNO Q에 모니터·키보드를 연결하고, 데스크톱의 Terminal 앱에서 직접 명령을 입력합니다.

  • 가장 단순. 보드 앞에서 바로 작업.
  • 한글 입력/표시가 데스크톱 환경에 따라 가능.
  • 대시보드 확인도 보드 브라우저로 http://localhost:8501 가능.
💡 언제 이 방법?모니터·키보드가 보드에 연결돼 있고, 부스에 보드를 두고 쓸 때. 별도 네트워크 설정이 필요 없어 학교망 제약과 무관합니다.

같은 WiFi의 PC에서 무선으로 보드에 접속해 작업합니다. SSH 창이 곧 보드 터미널이라, 입력 명령은 A와 동일합니다.

⚠️ 왜 SSH를 쓰나 — 한글 깨짐 회피시리얼 콘솔로 코드를 붙여넣으면 한글·°C·kΩ가 깨집니다(전송 경로 문제). SSH는 이게 안 깨져서, 한글판 코드를 그대로 붙여넣을 수 있습니다. (영문 ASCII판으로 작업하면 어느 방법이든 안 깨짐 — 패널 'EN' 참조)

① SSH 켜기 (보드에서 1회)

# 지금 켜기 + 부팅 시 자동 시작
sudo systemctl enable --now ssh

② 구동 확인

sudo systemctl status ssh          # active (running) + listening 0.0.0.0:22 확인 (q로 나옴)

③ PC에서 접속

PC가 보드와 같은 WiFi에 있어야 합니다. PowerShell 권장(Windows 기본 ssh).

ssh arduino@192.168.1.67           # 위 hostname -I 의 IP
# 비밀번호(보드 arduino 계정) 입력 → 접속되면 PC 터미널이 곧 보드 터미널
⚠️ WSL 말고 PowerShellWSL은 가상 네트워크라 보드에 못 닿는 경우가 많습니다(WSL의 hostname -I는 PC 실제 IP가 아님). PowerShell에서 하세요. PuTTY는 Translation을 UTF-8로.
💡 안 될 때ping 192.168.1.67로 먼저 확인 — 무응답이면 다른 WiFi(또는 학교망 격리). Connection reset이면 보드 SSH는 정상인데 망이 막는 것(학교망 격리) → 핫스팟 우회. 같은 192.168.x 대역인지 PC ipconfig로 확인.

④ 작업 끝나면 — SSH 끄기 (보안)

sudo systemctl disable --now ssh   # 지금 끄기 + 자동시작 해제
SSH vs 브라우저 — 통로가 다름SSH=터미널 제어(명령·파일), 대시보드=브라우저(http://IP:8501). SSH로는 그래픽을 못 봅니다. 둘은 같은 WiFi의 PC에서 동시에 띄워 씁니다.
✅ 정리둘 중 무엇을 골라도 이후 명령은 동일합니다. SSH는 한글 깨짐을 피하고 무선으로 편하게 하려는 선택지일 뿐, 필수는 아닙니다. 모니터가 있으면 A, PC에서 원격·한글판이 필요하면 B를 쓰세요. 이후 패널들(브로커·hub.py·대시보드·systemd)은 선택한 환경에서 그대로 진행하면 됩니다.

2허브 — 브로커 (로컬 / 클라우드 택일)

노드가 보낸 메시지를 받아줄 중계소(브로커)를 준비합니다. 두 방식 중 하나만 고르세요. 여기서 정한 브로커 주소가 곧 노드 펌웨어의 BROKER 값이 됩니다(다음 노드 패널에서 입력).

⚠️ 어느 걸 고를까 — 학교망이면 클라우드집·독립공유기 등 기기끼리 통신 자유로운 망이면 로컬이 간단합니다. 학교·기업 WiFi는 보안상 기기 간 통신을 막는 경우가 많아(클라이언트 격리), 로컬은 노드→허브가 rc=-2로 실패합니다. 그땐 클라우드로 우회하세요. (격리 진단법은 트러블슈팅 참조)

UNO Q 안에 mosquitto 브로커를 설치합니다. 허브 터미널에서 진행.

① 설치 (자동시작 등록까지 한 번에)

sudo apt update
sudo apt install -y mosquitto mosquitto-clients

② 외부 노드 접속 허용 설정

기본 설정은 UNO Q 자신(localhost)만 허용합니다. 다른 기기인 노드가 붙으려면 설정 파일에 두 줄을 추가해야 합니다.

핵심 — 실제 설정은 단 2줄listener 1883 0.0.0.0 = 모든 네트워크에서 1883포트 수신(노드 접속 허용) · allow_anonymous true = 사용자/암호 없이 접속(내부망 전용). 아래 두 방식 중 편한 쪽으로 /etc/mosquitto/conf.d/multinode_sensor_demo.conf 파일을 만드세요.
방법 A — heredoc (통째 복사, 주석 없이)

아래 전체를 복사해 터미널에 붙여넣으면 파일이 바로 만들어집니다. 한글 주석을 넣지 않습니다 — heredoc에 한글이 많으면 터미널 인코딩 충돌로 글자가 깨지기 때문입니다.

sudo tee /etc/mosquitto/conf.d/multinode_sensor_demo.conf > /dev/null << 'EOF'
listener 1883 0.0.0.0
allow_anonymous true
EOF
방법 B — nano (한글 주석 넣어도 안전)

설명을 한글 주석으로 남기고 싶다면 nano를 쓰세요. 에디터가 파일에 직접 저장하므로 한글이 깨지지 않습니다.

sudo nano -i /etc/mosquitto/conf.d/multinode_sensor_demo.conf

열리면 아래 내용 입력 → Ctrl+O → Enter → Ctrl+X:

# 멀티노드 환경 센싱 브로커 설정
# 노드(다른 기기)가 내부망으로 접속하도록 개방
listener 1883 0.0.0.0
allow_anonymous true
⚠️ heredoc에 한글 주석 금지지난 사고의 원인입니다. << 'EOF' 안에 한글 주석을 넣고 복붙하면 터미널이 한 글자씩 받아 처리하다 인코딩이 깨집니다. 한글 설명이 필요하면 heredoc 말고 nano(방법 B)로.

설정을 만든 뒤 적용을 위해 재시작:

sudo systemctl restart mosquitto
systemctl status mosquitto        # active (running) 확인 → q 로 빠져나옴
만든 파일 확인cat /etc/mosquitto/conf.d/multinode_sensor_demo.conf 로 내용이 제대로 들어갔는지 확인. 글자가 깨졌으면 방법 A로 다시 실행하면 덮어쓰기됩니다(tee는 기존 내용을 새로 씀).
보안 메모인증 없이 익명 접속을 허용하므로 내부망 전용 가정입니다. 학교 라우터에 외부 포트포워딩(1883)이 없어야 안전하며, 외부 노출이 필요하면 이후 username/password 또는 TLS를 추가합니다.
⚠️ 방화벽이 켜져 있다면대부분 UNO Q 기본은 방화벽(ufw)이 꺼져 있어 그대로 됩니다. 혹시 막힌다면 sudo ufw allow 1883 한 줄로 엽니다.
✅ 노드에 넣을 BROKER 값로컬은 허브(UNO Q)의 IP입니다. hostname -I로 확인한 192.168.x.x(또는 <보드명>.local), 포트 1883. 이 값을 다음 노드 펌웨어 패널의 로컬 탭 BROKER에 넣습니다.

━━ 0) 파이썬 환경 구축 — uv ━━

hub.py 전체 코드 — 로컬/클라우드 선택

🔀 브로커 방식 따라 다름노드 펌웨어와 같은 방식을 고르세요. 로컬은 localhost:1883, 클라우드는 HiveMQ:8883 + TLS. 저장 로직은 동일하고 접속 부분만 다릅니다.

로컬 브로커(localhost:1883)에서 수신. 실행: .venv/bin/python hub.py

hub.py 
"""
환경 센싱 Hub  —  MQTT subscriber + SQLite writer  (실수집용)
- 수신   : 로컬 mosquitto broker (localhost), 토픽 multinode_sensor_demo/+/bme688
- 저장   : SQLite (sensor_data.db)
- 페어   : dashboard.py(읽기 전용)
- 실행   : App Lab 앱으로 / 또는 systemd 서비스로 24/7 무인 운영
- 데이터 : 노드가 보낸 5분 평균 1건 = DB 1행
"""
import json, sqlite3, signal, sys
import paho.mqtt.client as mqtt

# ── 설정 ─────────────────────────────────────────
BROKER = "localhost"             # Q보드에서 직접 돌리므로 로컬 브로커
PORT   = 1883
TOPIC  = "multinode_sensor_demo/+/bme688"        # + = 모든 노드
DB     = "sensor_data.db"             # 실수집용 새 파일 (없으면 자동 생성)

# stdout 라인버퍼링 → systemd journalctl에 print가 즉시 보임
sys.stdout.reconfigure(line_buffering=True)

# ── DB 초기화 (확정 스키마) ─────────────────────
conn = sqlite3.connect(DB, check_same_thread=False)
conn.execute("""
CREATE TABLE IF NOT EXISTS readings(
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  ts    TEXT    DEFAULT CURRENT_TIMESTAMP,    -- 도착시각(UTC). 분석 시 +9h(KST)
  node  TEXT,
  temp  REAL,
  hum   REAL,
  press REAL,
  gas   REAL,
  n     INTEGER,                               -- 평균에 쓴 샘플 수(품질지표)
  co2   REAL                                   -- 추후 SCD30 라벨(없으면 NULL)
)""")
conn.commit()

# ── MQTT 콜백 (paho 2.x VERSION2 시그니처) ──────
def on_connect(client, userdata, flags, reason_code, properties):
    print(f"broker connect: {reason_code}")
    client.subscribe(TOPIC)

def on_message(client, userdata, msg):
    try:
        d = json.loads(msg.payload.decode())
    except Exception as e:
        print(f"parse 실패: {e}")
        return
    conn.execute(
        "INSERT INTO readings(node,temp,hum,press,gas,n) VALUES(?,?,?,?,?,?)",
        (d.get("node"), d.get("temp"), d.get("hum"),
         d.get("press"), d.get("gas"), d.get("n"))
    )
    conn.commit()
    print(f"저장: {d}")

# ── 우아한 종료 (systemd stop·재시작 시 SIGTERM) ─
def shutdown(signum, frame):
    print("종료 중...")
    try: client.disconnect()
    except Exception: pass
    try: conn.close()
    except Exception: pass
    sys.exit(0)

signal.signal(signal.SIGTERM, shutdown)
signal.signal(signal.SIGINT,  shutdown)

# ── MQTT 클라이언트 ─────────────────────────────
client = mqtt.Client(mqtt.CallbackAPIVersion.VERSION2)   # ★ deprecation 없음
client.on_connect = on_connect
client.on_message = on_message
client.connect(BROKER, PORT, keepalive=60)
client.loop_forever()

HiveMQ Cloud(8883 TLS)에서 수신. 상단 4개 값(Host·User·Pass)을 본인 정보로. 실행: .venv/bin/python hub_cloud.py

💡 추가 패키지 없음클라우드도 paho-mqtt 그대로 씁니다(TLS는 표준 라이브러리 ssl). uv 환경은 동일.
hub_cloud.py 
"""
Sensor Hub [CLOUD]  --  MQTT subscriber + SQLite writer
- broker    : HiveMQ Cloud (TLS 8883) -- bypasses school-WiFi client isolation
- NOTE      : separate file from local (mosquitto) version; use one or the other
- subscribe : HiveMQ Cloud broker, topic multinode_sensor_demo/+/bme688
- store     : SQLite (sensor_data.db)
- pair      : dashboard.py (read-only)
- run       : via App Lab app, or as a systemd service for 24/7 unattended operation
- data      : one averaged record from a node = one DB row
"""
import json, sqlite3, signal, sys, ssl     # ssl: for TLS (cloud)
import paho.mqtt.client as mqtt

# ---- Config (HiveMQ Cloud) ----
BROKER   = "xxxxx.s1.eu.hivemq.cloud"   # HiveMQ Overview Host
PORT     = 8883                          # TLS port (local mosquitto was 1883)
USERNAME = "여기_username"               # HiveMQ Access Management
PASSWORD = "여기_password"
TOPIC    = "multinode_sensor_demo/+/bme688"        # + = all nodes
DB       = "sensor_data.db"             # data file (auto-created if absent)

# line-buffered stdout -> print() shows up immediately in journalctl
sys.stdout.reconfigure(line_buffering=True)

# ---- DB init (schema) ----
conn = sqlite3.connect(DB, check_same_thread=False)
conn.execute("""
CREATE TABLE IF NOT EXISTS readings(
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  ts    TEXT    DEFAULT CURRENT_TIMESTAMP,    -- time (UTC); +9h for KST in analysis
  node  TEXT,
  temp  REAL,
  hum   REAL,
  press REAL,
  gas   REAL,
  n     INTEGER,                               -- sample count in the average (quality)
  co2   REAL                                   -- optional future SCD30 label (NULL if unused)
)""")
conn.commit()

# ---- MQTT callbacks (paho 2.x VERSION2 signature) ----
def on_connect(client, userdata, flags, reason_code, properties):
    print(f"broker connect: {reason_code}")
    client.subscribe(TOPIC)

def on_message(client, userdata, msg):
    try:
        d = json.loads(msg.payload.decode())
    except Exception as e:
        print(f"parse failed: {e}")
        return
    # If measurement time (t, UTC) is present, store it as ts (node NTP sync).
    # If absent (NTP-fail fallback), omit ts -> DB fills arrival time (CURRENT_TIMESTAMP).
    t = d.get("t")
    if t:
        conn.execute(
            "INSERT INTO readings(ts,node,temp,hum,press,gas,n) VALUES(?,?,?,?,?,?,?)",
            (t, d.get("node"), d.get("temp"), d.get("hum"),
             d.get("press"), d.get("gas"), d.get("n")))
    else:
        conn.execute(
            "INSERT INTO readings(node,temp,hum,press,gas,n) VALUES(?,?,?,?,?,?)",
            (d.get("node"), d.get("temp"), d.get("hum"),
             d.get("press"), d.get("gas"), d.get("n")))
    conn.commit()
    print(f"saved: {d}")

# ---- graceful shutdown (SIGTERM on systemd stop/restart) ----
def shutdown(signum, frame):
    print("shutting down...")
    try: client.disconnect()
    except Exception: pass
    try: conn.close()
    except Exception: pass
    sys.exit(0)

signal.signal(signal.SIGTERM, shutdown)
signal.signal(signal.SIGINT,  shutdown)

# ---- MQTT client (TLS + auth for cloud) ----
client = mqtt.Client(mqtt.CallbackAPIVersion.VERSION2)   # no deprecation warning
client.username_pw_set(USERNAME, PASSWORD)               # cloud: login
client.tls_set(tls_version=ssl.PROTOCOL_TLS_CLIENT)      # cloud: enable TLS
# client.tls_insecure_set(True)   # uncomment to skip cert verification (matches node setInsecure)
client.on_connect = on_connect
client.on_message = on_message
client.connect(BROKER, PORT, keepalive=60)
client.loop_forever()
⚠️ systemd로 클라우드 쓸 때서비스 파일 ExecStart의 hub.py를 hub_cloud.py로 바꾸고 daemon-reload → restart.

4노드 — 구성 (하드웨어)

허브 준비(패널 1~3)가 끝났으면 이제 노드 차례입니다. 구성 → IDE → 펌웨어까지 한 번에 진행하고, 펌웨어에는 패널 2에서 정한 브로커 주소를 넣습니다. 기본 기종은 UNO R4 WiFi + Grove Base Shield + BME688, Grove 커넥터만 체결하면 됩니다(납땜 없음).

⚠️ STEP 0 — 전압 스위치 5V (UNO R4 기준)Grove Base Shield의 VCC 스위치를 5V로 두세요. UNO R4 WiFi는 5V 동작입니다. (대안 기종 Nano ESP32를 쓸 땐 반대로 3.3V — 기종에 맞게)

구성 순서 (UNO R4 WiFi)

  • ① Grove Base Shield를 R4 위에 장착 — 핀 헤더 방향을 맞춰 R4 위에 그대로 얹어 꽂기

  • ④ USB-C 케이블로 PC 연결 — 펌웨어 업로드용
💡 I2C 포트를 쓰는 이유BME688은 I2C 방식 센서입니다. 실드의 I2C 포트는 보통 표시돼 있어 헷갈릴 일이 적습니다. Grove 커넥터는 한 방향으로만 꽂히니 억지로 밀지 마세요.
확인용 체크BME688 모듈에 LED가 있다면 전원 연결 시 불이 들어오는지 보세요. 안 들어오면 전압 스위치(R4=5V)·케이블·I2C 포트를 다시 확인.
대안 기종 — Nano ESP32R4 대신 Nano ESP32를 쓰면: Grove Shield for Nano에 ESP32 장착, 전압 스위치는 3.3V(R4와 반대), 전원은 USB-C. 나머지(BME688 I2C 체결)는 동일. 펌웨어는 패널 6의 ESP32 탭.

/*
 * ============================================================
 *  멀티노드 환경 센싱 펌웨어  ★ Arduino UNO R4 WiFi 판
 *  (Nano ESP32 판과 동일 동작 — WiFi/NTP 부분만 R4 방식)
 *  - 센서 : Grove BME688 (I2C 0x76)  ← Grove 실드의 I2C 포트
 *  - 전송 : WiFi -> MQTT over TLS (HiveMQ Cloud 브로커, 8883)
 *  - 시각 : NTP(WiFi.getTime, UTC) → 정각 격자 측정·발행
 *  - 방식 : SAMPLE_MS 마다 샘플 → PUBLISH_MIN 분 평균 → 정각 발행
 *           페이로드에 측정시각(t, UTC) 포함 → 노드 간 정렬
 * ============================================================
 *  ⚠️ 하드웨어 주의 — Nano ESP32 와 반대!
 *    UNO R4 WiFi 는 [5V] 보드 → Grove Shield 전압 스위치를 [5V] 로!
 *    (Nano ESP32 는 3.3V 였지만, R4 는 5V 입니다)
 *  전원 : USB-C 또는 배럴잭(DC, 7~12V, KC인증 아두이노 전용 어댑터) 사용 가능
 * ============================================================
 *  보드 매니저:  "Arduino UNO R4 Boards"  (보드: Arduino UNO R4 WiFi)
 *  필요 라이브러리:
 *    - PubSubClient            (by Nick O'Leary)
 *  [클라우드 브로커 버전] HiveMQ Cloud(TLS 8883). 로컬 버전과 별개 파일.
 *    - Adafruit BME680 Library (+ Adafruit Unified Sensor 동반)
 *    ※ WiFiS3 는 UNO R4 보드 패키지에 내장 — 따로 설치 안 함
 * ============================================================
 */

#include <WiFiS3.h>          // ★ R4 WiFi 전용 (ESP32의 WiFi.h 아님)
#include <PubSubClient.h>
#include <Wire.h>
#include "Adafruit_BME680.h"
#include <time.h>            // gmtime/strftime (측정시각 문자열 변환)

// ===================== 사용자 설정 =====================
const char* WIFI_SSID = "your-wifi-ssid";      // ★ 현장 WiFi 이름 (영문 권장)
const char* WIFI_PASS = "your-wifi-password";  // ★ WiFi 암호

// ★ HiveMQ Cloud 접속 정보 (Manage Cluster 에서 확인/생성)
const char* BROKER    = "xxxxx.s1.eu.hivemq.cloud";  // Overview 의 Host (끝 공백 주의!)
const int   PORT      = 8883;                         // TLS 포트 (로컬은 1883)
const char* MQTT_USER = "여기_username";              // Access Management 에서 생성
const char* MQTT_PASS = "여기_password";

// ★ 발행 주기(분). 1 / 5 / 10 등. 모든 노드를 같은 값으로!
const int           PUBLISH_MIN = 5;
const unsigned long SAMPLE_MS   = 10000;       // 샘플 간격(10초)
// ======================================================

WiFiSSLClient net;             // ★ 클라우드: R4 TLS 클라이언트 (로컬은 WiFiClient)
PubSubClient  client(net);
Adafruit_BME680 bme;

String nodeId, topic;
double  sT=0, sH=0, sP=0, sG=0;  int n = 0;
bool    bmeOK = false;
bool    timeOK = false;                          // NTP→RTC 동기화 성공 여부
long    curBucket = -1;
unsigned long lastSample = 0;

// nodeId = MAC 끝 3바이트
void makeNodeId() {
  byte mac[6];
  WiFi.macAddress(mac);
  char id[16];
  // R4 WiFi.macAddress 는 mac[0]이 끝 바이트인 구현이 있어 역순 주의.
  // 보드마다 고유하기만 하면 되므로 끝 3바이트 사용.
  snprintf(id, sizeof(id), "node_%02X%02X%02X", mac[2], mac[1], mac[0]);
  nodeId = String(id);
  topic  = "multinode_sensor_demo/" + nodeId + "/bme688";
}

void connectWiFi() {
  if (WiFi.status() == WL_CONNECTED) return;
  Serial.print("WiFi");
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  unsigned long t0 = millis();
  while (WiFi.status() != WL_CONNECTED && millis() - t0 < 15000) {
    delay(500); Serial.print(".");
  }
  Serial.println(WiFi.status() == WL_CONNECTED ? " OK" : " FAIL");
}

void connectBroker() {
  while (!client.connected()) {
    connectWiFi();
    String cid = nodeId + "-" + String(random(0xffff), HEX);
    Serial.print("MQTT");
    if (client.connect(cid.c_str(), MQTT_USER, MQTT_PASS)) Serial.println(" OK");  // ★ 인증
    else { Serial.print(" rc="); Serial.println(client.state()); delay(2000); }
  }
}

// NTP(UTC epoch) 동기화 확인. R4 WiFi는 WiFi.getTime()이 현재 UTC epoch 반환.
// → RTC 객체에 굳이 안 넣고 WiFi.getTime()을 직접 시각원으로 사용(단순·안전).
//   (RTCTime 생성자/분해 API의 보드패키지 버전차를 피함)
void syncTime() {
  Serial.print("NTP 동기화");
  unsigned long epoch = 0;
  unsigned long t0 = millis();
  while ((epoch = WiFi.getTime()) == 0 && millis() - t0 < 10000) {
    Serial.print("."); delay(500);
  }
  if (epoch > 0) {
    timeOK = true;
    Serial.print(" OK (UTC epoch "); Serial.print(epoch); Serial.println(")");
  } else {
    timeOK = false;
    Serial.println(" FAIL -> 정각정렬 없이 동작(측정시각은 허브 도착시각 사용)");
  }
}

// 현재 epoch초 (UTC) — WiFi.getTime() 직접 사용
long nowEpoch() {
  unsigned long e = WiFi.getTime();
  return (long)e;
}

// 버킷 epoch초 → "YYYY-MM-DD HH:MM:SS" (UTC)
// RTCTime 분해 API(버전차) 대신 표준 C gmtime 사용 → 안전·이식적
String epochToStr(long epoch) {
  time_t t = (time_t)epoch;
  struct tm *tm_utc = gmtime(&t);
  char buf[32];
  strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S", tm_utc);
  return String(buf);
}

void publishAverage(long bucketEpoch) {
  if (n <= 0) return;
  char p[220];
  if (timeOK) {
    String ts = epochToStr(bucketEpoch);
    snprintf(p, sizeof(p),
      "{\"node\":\"%s\",\"t\":\"%s\",\"temp\":%.2f,\"hum\":%.2f,"
      "\"press\":%.2f,\"gas\":%lu,\"n\":%d}",
      nodeId.c_str(), ts.c_str(), sT/n, sH/n, sP/n,
      (unsigned long)(sG/n), n);
  } else {
    snprintf(p, sizeof(p),
      "{\"node\":\"%s\",\"temp\":%.2f,\"hum\":%.2f,"
      "\"press\":%.2f,\"gas\":%lu,\"n\":%d}",
      nodeId.c_str(), sT/n, sH/n, sP/n, (unsigned long)(sG/n), n);
  }
  client.publish(topic.c_str(), p);
  Serial.print("PUB: "); Serial.println(p);
}

void takeSample() {
  if (bmeOK && bme.performReading()) {
    sT += bme.temperature;
    sH += bme.humidity;
    sP += bme.pressure / 100.0;     // Pa -> hPa
    sG += bme.gas_resistance;       // Ohm
    n++;
  }
}

void resetAccum() { sT=sH=sP=sG=0; n=0; }

void setup() {
  Serial.begin(115200);
  delay(300);

  Wire.begin();
  Wire.setClock(100000);

  bmeOK = bme.begin(0x76);
  if (bmeOK) {
    bme.setTemperatureOversampling(BME680_OS_8X);
    bme.setHumidityOversampling(BME680_OS_2X);
    bme.setPressureOversampling(BME680_OS_4X);
    bme.setIIRFilterSize(BME680_FILTER_SIZE_3);
    bme.setGasHeater(320, 150);
  } else {
    Serial.println("BME688 not found @0x76 - 배선/전압스위치(5V) 확인");
  }

  connectWiFi();
  makeNodeId();
  syncTime();                // ★ NTP → RTC
  client.setServer(BROKER, PORT);
  connectBroker();

  Serial.print("Node: ");  Serial.println(nodeId);
  Serial.print("Topic: "); Serial.println(topic);
  Serial.print("발행주기: "); Serial.print(PUBLISH_MIN); Serial.println("분");

  if (timeOK) {
    long sec = PUBLISH_MIN * 60L;
    curBucket = (nowEpoch() / sec) * sec;
  }
}

void loop() {
  if (!client.connected()) connectBroker();
  client.loop();

  unsigned long now = millis();
  long sec = PUBLISH_MIN * 60L;

  if (timeOK) {
    // ── 정각 정렬 모드 (NTP 성공) ──
    long epoch  = nowEpoch();
    long bucket = (epoch / sec) * sec;

    if (curBucket < 0) curBucket = bucket;
    if (bucket != curBucket) {
      publishAverage(curBucket);
      resetAccum();
      curBucket = bucket;
    }
    if (now - lastSample >= SAMPLE_MS) {
      lastSample = now; takeSample();
      Serial.print("sample "); Serial.print(n);
      Serial.print(" (bucket "); Serial.print(epochToStr(curBucket)); Serial.println(")");
    }
  } else {
    // ── fallback (NTP 실패) — millis 기준 주기 발행 ──
    static unsigned long lastPub = 0;
    if (now - lastSample >= SAMPLE_MS) {
      lastSample = now; takeSample();
      Serial.print("sample "); Serial.print(n); Serial.println(" (no-NTP)");
    }
    if (now - lastPub >= (unsigned long)PUBLISH_MIN * 60000UL) {
      lastPub = now; publishAverage(0); resetAccum();
    }
  }
}

void connectWiFi() {
  // 이미 연결 + 유효 IP 있으면 통과
  if (WiFi.status() == WL_CONNECTED && WiFi.localIP() != IPAddress(0,0,0,0)) return;

// NTP 서버 (인터넷 필요). 학교망이 막으면 시각동기화 실패 → fallback 동작.
const char* NTP1 = "pool.ntp.org";
const char* NTP2 = "time.google.com";
// ======================================================

WiFiClient   net;
PubSubClient client(net);
Adafruit_BME680 bme;        // I2C

String nodeId, topic;
double  sT=0, sH=0, sP=0, sG=0;  int n = 0;     // 평균 누적기
bool    bmeOK = false;
bool    timeOK = false;                          // NTP 동기화 성공 여부
long    curBucket = -1;                          // 현재 측정 중인 정각 버킷(epoch초)
unsigned long lastSample = 0;

// nodeId = MAC 끝 3바이트 → 보드마다 고유
void makeNodeId() {
  uint8_t mac[6];
  WiFi.macAddress(mac);
  char id[16];
  snprintf(id, sizeof(id), "node_%02X%02X%02X", mac[3], mac[4], mac[5]);
  nodeId = String(id);
  topic  = "multinode_sensor_demo/" + nodeId + "/bme688";
}

void connectWiFi() {
  // 이미 연결 + 유효 IP 있으면 통과
  if (WiFi.status() == WL_CONNECTED && WiFi.localIP() != IPAddress(0,0,0,0)) return;

  Serial.print("WiFi");
  WiFi.disconnect();           // 깨끗하게 재연결 (밤사이 끊김 복구에 도움)
  delay(100);
  WiFi.begin(WIFI_SSID, WIFI_PASS);

  // 1단계: WiFi 인증(연결)까지 대기 (최대 15초)
  unsigned long t0 = millis();
  while (WiFi.status() != WL_CONNECTED && millis() - t0 < 15000) {
    delay(500); Serial.print(".");
  }
  if (WiFi.status() != WL_CONNECTED) {
    Serial.println(" FAIL (인증 실패 - SSID/암호/신호 확인)");
    return;
  }
  Serial.println(" OK");

  // 2단계: ★ 유효 IP(DHCP) 받을 때까지 대기 (최대 12초)
  //   WiFi는 붙어도 IP가 0.0.0.0이면 NTP/MQTT 전부 실패 → 이 대기가 핵심
  Serial.print("  IP 획득");
  unsigned long t1 = millis();
  while (WiFi.localIP() == IPAddress(0,0,0,0) && millis() - t1 < 12000) {
    delay(500); Serial.print("+");
  }
  IPAddress ip = WiFi.localIP();
  if (ip == IPAddress(0,0,0,0)) {
    Serial.println(" FAIL (DHCP IP 미수신 - 라우터/망 문제)");
  } else {
    Serial.println(" OK");
  }
  // 진단 출력: IP/게이트웨이/DNS (0.0.0.0이면 망 문제)
  Serial.print("  IP : "); Serial.println(ip);
  Serial.print("  GW : "); Serial.println(WiFi.gatewayIP());
  Serial.print("  DNS: "); Serial.println(WiFi.dnsIP());
  Serial.print("  RSSI: "); Serial.print(WiFi.RSSI()); Serial.println(" dBm (신호; -50좋음 -80약함)");
}

void connectBroker() {
  while (!client.connected()) {
    connectWiFi();
    String cid = nodeId + "-" + String(random(0xffff), HEX);
    Serial.print("MQTT");
    if (client.connect(cid.c_str())) Serial.println(" OK");
    else { Serial.print(" rc="); Serial.println(client.state()); delay(2000); }
  }
}

// NTP 동기화 (UTC). 성공하면 timeOK=true.
void syncTime() {
  configTime(0, 0, NTP1, NTP2);        // 0,0 = UTC (오프셋 0). 분석은 UTC 기준
  Serial.print("NTP 동기화");
  struct tm tinfo;
  unsigned long t0 = millis();
  while (!getLocalTime(&tinfo, 500) && millis() - t0 < 10000) {
    Serial.print(".");
  }
  timeOK = getLocalTime(&tinfo, 500);
  if (timeOK) {
    char buf[32]; strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S", &tinfo);
    Serial.print(" OK (UTC "); Serial.print(buf); Serial.println(")");
  } else {
    Serial.println(" FAIL -> 정각정렬 없이 동작(측정시각은 허브 도착시각 사용)");
  }
}

long nowEpoch() { time_t now; time(&now); return (long)now; }

// 버킷 epoch초 → "YYYY-MM-DD HH:MM:SS" (UTC)
String epochToStr(long epoch) {
  time_t t = (time_t)epoch; struct tm tinfo;
  gmtime_r(&t, &tinfo);
  char buf[32]; strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S", &tinfo);
  return String(buf);
}

void publishAverage(long bucketEpoch) {
  if (n <= 0) return;
  char p[220];
  if (timeOK) {
    String ts = epochToStr(bucketEpoch);   // 측정시각 = 구간 시작 정각(UTC)
    snprintf(p, sizeof(p),
      "{\"node\":\"%s\",\"t\":\"%s\",\"temp\":%.2f,\"hum\":%.2f,"
      "\"press\":%.2f,\"gas\":%lu,\"n\":%d}",
      nodeId.c_str(), ts.c_str(), sT/n, sH/n, sP/n,
      (unsigned long)(sG/n), n);
  } else {
    snprintf(p, sizeof(p),                 // NTP 실패 → t 생략(허브가 도착시각)
      "{\"node\":\"%s\",\"temp\":%.2f,\"hum\":%.2f,"
      "\"press\":%.2f,\"gas\":%lu,\"n\":%d}",
      nodeId.c_str(), sT/n, sH/n, sP/n, (unsigned long)(sG/n), n);
  }
  client.publish(topic.c_str(), p);
  Serial.print("PUB: "); Serial.println(p);
}

void takeSample() {
  if (bmeOK && bme.performReading()) {
    sT += bme.temperature;
    sH += bme.humidity;
    sP += bme.pressure / 100.0;        // Pa -> hPa
    sG += bme.gas_resistance;          // Ohm
    n++;
  }
}

void resetAccum() { sT=sH=sP=sG=0; n=0; }

void setup() {
  Serial.begin(115200);
  delay(300);

  Wire.begin();              // Grove I2C 자동 매핑
  Wire.setClock(100000);     // 다중장치 안정성 위해 100kHz

  bmeOK = bme.begin(0x76);   // Grove BME688 = 0x76
  if (bmeOK) {
    bme.setTemperatureOversampling(BME680_OS_8X);
    bme.setHumidityOversampling(BME680_OS_2X);
    bme.setPressureOversampling(BME680_OS_4X);
    bme.setIIRFilterSize(BME680_FILTER_SIZE_3);
    bme.setGasHeater(320, 150);
  } else {
    Serial.println("BME688 not found @0x76 - 배선/전압스위치(3.3V) 확인");
  }

  connectWiFi();
  makeNodeId();
  syncTime();                // ★ NTP 동기화 (UTC)
  client.setServer(BROKER, PORT);
  connectBroker();

  Serial.print("Node: ");  Serial.println(nodeId);
  Serial.print("Topic: "); Serial.println(topic);
  Serial.print("발행주기: "); Serial.print(PUBLISH_MIN); Serial.println("분");

  if (timeOK) {
    long sec = PUBLISH_MIN * 60L;
    curBucket = (nowEpoch() / sec) * sec;   // 현재 속한 정각 버킷
  }
}

void loop() {
  // WiFi가 끊겼거나 IP를 잃으면 먼저 재연결 (밤사이 망 변동 복구)
  if (WiFi.status() != WL_CONNECTED || WiFi.localIP() == IPAddress(0,0,0,0)) {
    connectWiFi();
  }
  if (!client.connected()) connectBroker();
  client.loop();             // MQTT 연결 유지

  unsigned long now = millis();
  long sec = PUBLISH_MIN * 60L;

  if (timeOK) {
    // ── 정각 정렬 모드 (NTP 성공) ──
    long epoch  = nowEpoch();
    long bucket = (epoch / sec) * sec;       // 지금 속한 정각 버킷

    if (curBucket < 0) curBucket = bucket;
    if (bucket != curBucket) {               // 정각 경과 → 직전 버킷 발행
      publishAverage(curBucket);
      resetAccum();
      curBucket = bucket;
    }
    if (now - lastSample >= SAMPLE_MS) {
      lastSample = now; takeSample();
      Serial.printf("sample %d (bucket %s)\n", n, epochToStr(curBucket).c_str());
    }
  } else {
    // ── fallback 모드 (NTP 실패) — millis 기준 주기 발행 ──
    static unsigned long lastPub = 0;
    if (now - lastSample >= SAMPLE_MS) {
      lastSample = now; takeSample();
      Serial.printf("sample %d (no-NTP)\n", n);
    }
    if (now - lastPub >= (unsigned long)PUBLISH_MIN * 60000UL) {
      lastPub = now; publishAverage(0); resetAccum();
    }
  }
}
⏱️ 시각 동기화(NTP) & 정각 정렬 — 공통 노드마다 켠 시각이 달라도 NTP로 인터넷 시각을 받아 정각 격자에 맞춰 측정·발행합니다(로컬·클라우드 공통).
  • PUBLISH_MIN = 발행 주기(분). 이 값이 정각 격자. ★ 모든 노드를 같은 값으로.
  • 샘플은 SAMPLE_MS(10초) 간격, 발행은 PUBLISH_MIN분 평균(1분=6개, 5분=30개).
  • 측정시각 "t"(UTC)를 페이로드에 포함 → 허브가 DB ts로 저장 → 분석 정렬 용이.
⚠️ NTP는 인터넷 필요 — 안 되면 자동 fallback학교망이 NTP를 막으면 시각동기화 실패 → 정각정렬 없이 주기 발행(측정시각은 허브 도착시각). 시리얼에 NTP 동기화 OK/FAIL 표시. NTP 실패해도 발행은 계속됩니다(곁가지). 진짜 수집 실패는 MQTT rc=-2가 원인.

        return pd.DataFrame()
    sql = ("SELECT datetime(ts,'+9 hours') AS recv_time, node, "
           "temp, hum, press, gas/1000.0 AS gas "
           "FROM readings ORDER BY id DESC LIMIT ?")
    with closing(sqlite3.connect(DB)) as con:
        df = pd.read_sql_query(sql, con, params=(limit,))
    return df.iloc[::-1].reset_index(drop=True) if not df.empty else df

def half_gauge(key: str, value) -> go.Figure:

⚠️ ★ 브로커 주소 — "localhost"가 안 통한다App Lab은 main.py를 Docker 컨테이너로 실행합니다. 컨테이너 안의 localhost는 컨테이너 자신을 가리켜, 호스트(UNO Q)에서 도는 mosquitto에 안 닿습니다(Connection refused). 또 .local(mDNS)도 컨테이너에선 못 풉니다(Name or service not known).

해결 — 호스트의 Docker 게이트웨이 IP를 직접 기입:

# 1) 실행 중인 컨테이너 이름 확인
docker ps

# 2) 컨테이너에서 본 호스트(게이트웨이) IP 확인
docker exec <컨테이너이름> ip route | grep default
# → "default via 172.19.0.1 ..." 처럼 나오는 그 IP

# 3) 접속 테스트 (OK 나오면 그 IP가 정답)
docker exec <컨테이너이름> python3 -c "import socket; socket.create_connection(('172.19.0.1',1883),3); print('OK')"
그 IP를 main.py의 BROKER에 기입 (예: BROKER = "172.19.0.1"). 따옴표 안에 공백이 들어가지 않도록 주의("172.19.0.1 "처럼 끝 공백이 있으면 연결 실패).
⚠️ IP가 바뀔 수 있음 — mDNS보다 IP 확인이 안전이 게이트웨이 IP(172.19.0.1 등)는 App Lab이 컨테이너를 재생성하면 대역이 바뀔 수 있습니다(172.19 → 172.20 등). .local mDNS는 컨테이너에서 아예 안 되므로, 막히면 위 ip route로 현재 IP를 다시 확인해 기입하세요. 장기 무인 수집이 목적이면 애초에 트랙 B(systemd)가 정답입니다 — 컨테이너가 아니라 localhost가 바로 통하고 IP 변동도 없습니다.
💡 로그 확인 — App Lab 콘솔이 아니라 docker logsApp Lab 콘솔은 컨테이너 시작까지만 보여주고, 파이썬 print([수집] 등)는 안 보입니다. 실제 동작은 터미널에서: 
docker logs --tail 30 <컨테이너이름>
[수집] broker connect: Success + [수집] 저장: {...} 가 보이면 정상.
⚠️ App Lab의 한계 — Run 버튼App Lab 자체는 부팅 시 열리지만, 앱은 사람이 Run을 눌러야 시작됩니다. "전원만 켜면 무인 수집"은 App Lab만으로는 안 됩니다 — 그 용도는 트랙 B(systemd).

━━ 트랙 B: systemd (무인 자동 수집) ━━

전원만 켜면 수집·표시가 자동으로 돌게 만듭니다. 모니터·키보드 없이 PC에서 SSH로 관리합니다. 컨테이너가 아니라 호스트에서 직접 실행하므로 BROKER="localhost"가 그대로 통하고 IP 변동 문제도 없습니다.

?개념 정리 (FAQ)

원리를 알면 응용·확장이 쉽습니다.

왜 노드에서 5분 평균을 보내나?

가스 저항값은 노이즈·드리프트가 큽니다. 10초마다 샘플을 모아 5분 평균 1건만 보내면, 노이즈가 줄고 네트워크 트래픽도 1/30로 감소합니다. 평균에 쓴 샘플 수(n)도 함께 보내 품질 지표로 씁니다.

MQTT의 발행/구독이 뭔가?

노드는 자기 토픽으로 발행만 하고 누가 받는지 모릅니다. hub.py는 multinode_sensor_demo/+/bme688(+=아무 노드)을 구독해 모든 노드를 한 번에 받습니다. 노드를 추가해도 hub.py 수정이 필요 없는 이유입니다.

App Lab과 systemd, 뭐가 다른가?

App Lab은 사람이 Run을 누르는 개발·시연 환경, systemd는 OS가 부팅 때 자동으로 띄우는 운영 환경입니다. mosquitto는 apt 설치만으로 이미 systemd에 등록돼 늘 떠 있습니다.

DB 용량이 걱정되는데?

8노드 × 5분 간격이면 하루 약 2,300행. 1년 모아도 수십 MB 수준이라 충분합니다. ls -lh sensor_data.db로 점검할 수 있습니다.

도착 시각(ts)은 정확한가?

노드엔 시계가 없어, hub.py가 받은 시각(UTC)을 기록합니다. 대시보드는 읽을 때 +9시간(KST)으로 변환합니다. 분 단위 해상도엔 충분합니다.

hub.py (영문 ASCII) — 수집

MQTT 구독 → SQLite 저장. .venv/bin/python hub.py로 실행.

hub.py 
"""
Sensor Hub  --  MQTT subscriber + SQLite writer
- subscribe : local mosquitto broker (localhost), topic multinode_sensor_demo/+/bme688
- store     : SQLite (sensor_data.db)
- pair      : dashboard.py (read-only)
- run       : via App Lab app, or as a systemd service for 24/7 unattended operation
- data      : one averaged record from a node = one DB row
"""
import json, sqlite3, signal, sys
import paho.mqtt.client as mqtt

# ---- Config ----
BROKER = "localhost"             # runs on the Q board itself -> local broker
PORT   = 1883
TOPIC  = "multinode_sensor_demo/+/bme688"        # + = all nodes
DB     = "sensor_data.db"             # data file (auto-created if absent)

# line-buffered stdout -> print() shows up immediately in journalctl
sys.stdout.reconfigure(line_buffering=True)

# ---- DB init (schema) ----
conn = sqlite3.connect(DB, check_same_thread=False)
conn.execute("""
CREATE TABLE IF NOT EXISTS readings(
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  ts    TEXT    DEFAULT CURRENT_TIMESTAMP,    -- time (UTC); +9h for KST in analysis
  node  TEXT,
  temp  REAL,
  hum   REAL,
  press REAL,
  gas   REAL,
  n     INTEGER,                               -- sample count in the average (quality)
  co2   REAL                                   -- optional future SCD30 label (NULL if unused)
)""")
conn.commit()

# ---- MQTT callbacks (paho 2.x VERSION2 signature) ----
def on_connect(client, userdata, flags, reason_code, properties):
    print(f"broker connect: {reason_code}")
    client.subscribe(TOPIC)

def on_message(client, userdata, msg):
    try:
        d = json.loads(msg.payload.decode())
    except Exception as e:
        print(f"parse failed: {e}")
        return
    # If measurement time (t, UTC) is present, store it as ts (node NTP sync).
    # If absent (NTP-fail fallback), omit ts -> DB fills arrival time (CURRENT_TIMESTAMP).
    t = d.get("t")
    if t:
        conn.execute(
            "INSERT INTO readings(ts,node,temp,hum,press,gas,n) VALUES(?,?,?,?,?,?,?)",
            (t, d.get("node"), d.get("temp"), d.get("hum"),
             d.get("press"), d.get("gas"), d.get("n")))
    else:
        conn.execute(
            "INSERT INTO readings(node,temp,hum,press,gas,n) VALUES(?,?,?,?,?,?)",
            (d.get("node"), d.get("temp"), d.get("hum"),
             d.get("press"), d.get("gas"), d.get("n")))
    conn.commit()
    print(f"saved: {d}")

# ---- graceful shutdown (SIGTERM on systemd stop/restart) ----
def shutdown(signum, frame):
    print("shutting down...")
    try: client.disconnect()
    except Exception: pass
    try: conn.close()
    except Exception: pass
    sys.exit(0)

signal.signal(signal.SIGTERM, shutdown)
signal.signal(signal.SIGINT,  shutdown)

# ---- MQTT client ----
client = mqtt.Client(mqtt.CallbackAPIVersion.VERSION2)   # no deprecation warning
client.on_connect = on_connect
client.on_message = on_message
client.connect(BROKER, PORT, keepalive=60)
client.loop_forever()

    with closing(sqlite3.connect(DB)) as con:
        return pd.read_sql_query(sql, con)


def query_range(start_kst: datetime, end_kst: datetime) -> pd.DataFrame:
    """For date-range export. DB stores UTC; subtract 9h from KST range."""
    if not os.path.isfile(DB):
        return pd.DataFrame()
    start_utc = (start_kst - timedelta(hours=9)).strftime("%Y-%m-%d %H:%M:%S")
    end_utc   = (end_kst   - timedelta(hours=9)).strftime("%Y-%m-%d %H:%M:%S")
    sql = ("SELECT datetime(ts,'+9 hours') AS recv_time_kst, node, "
           "temp, hum, press, gas/1000.0 AS gas_kohm "
           "FROM readings WHERE ts BETWEEN ? AND ? ORDER BY id")
    with closing(sqlite3.connect(DB)) as con:
        return pd.read_sql_query(sql, con, params=(start_utc, end_utc))


def get_time_bounds():
    """Min/max time in DB (KST) -- defaults for the range picker."""
    if not os.path.isfile(DB):
        return None, None
    sql = ("SELECT MIN(datetime(ts,'+9 hours')), MAX(datetime(ts,'+9 hours')) "
           "FROM readings")
    with closing(sqlite3.connect(DB)) as con:
        lo, hi = con.execute(sql).fetchone()
    if not lo:
        return None, None
    return (datetime.fromisoformat(lo), datetime.fromisoformat(hi))

    _, _, _, gmin, gmax = METRICS[key]
    if gmax == gmin or pd.isna(value):

━━ (가) 운영 일시 종료 · 재개 ━━

데모가 끝났거나 잠깐 쉴 때. 삭제가 아니라 멈춤이라, 언제든 다시 켜집니다.

잠깐 멈추기 (설정은 유지) systemd로 운영 중이라면 — 자동시작 설정은 남기고 지금만 멈춤:

다시 켜기: 
sudo systemctl start multinode_sensor_demo_hub multinode_sensor_demo_dashboard
App Lab으로 운영 중이라면 — App Lab에서 Stop 버튼.
완전히 끄기 (전원 내리기 전) 데모를 끝내고 보드 전원을 뽑을 거라면 — 그냥 뽑아도 DB는 파일이라 안전하지만, 깔끔하게 끄려면: 
# 1) 서비스 정지 (수집 중 갑작스런 전원차단 방지)
sudo systemctl stop multinode_sensor_demo_hub multinode_sensor_demo_dashboard

# 2) 보드 안전 종료 (선택 — 파일시스템 보호)
sudo poweroff
전원만 뽑아도 큰 문제는 없지만, poweroff로 끄면 SD/eMMC 손상 위험이 줄어듭니다.
💡 자동시작 일시 해제 vs 완전 정지stop은 "지금만 멈춤"(다음 부팅 때 또 자동시작). 부팅 시 자동시작을 막고 싶으면 disable까지: sudo systemctl disable --now ..._hub ..._dashboard. 다시 자동운영하려면 enable --now.

━━ (나) DB만 초기화하고 재수집 ━━

테스트 데이터를 버리고 깨끗이 다시 모으고 싶을 때. 먼저 수집을 멈추세요(sudo systemctl stop multinode_sensor_demo_hub). 그다음 세 방법 중 하나:

방법 ① DB 파일 통째 삭제 (완전 초기화) 가장 확실. 다음 실행 때 빈 DB가 새로 생기고 id도 1부터. 
# WAL 모드라 파일이 3개 — 반드시 함께 삭제
cd ~/multinode_sensor_demo
rm -f sensor_data.db sensor_data.db-wal sensor_data.db-shm

# 다시 수집 시작 (빈 DB 자동 생성)
sudo systemctl start multinode_sensor_demo_hub multinode_sensor_demo_dashboard
⚠️ sensor_data.db만 지우면 안 됨 — WAL(-wal)에 남은 데이터가 되살아날 수 있어 3개 다 삭제.
방법 ② 테이블 내용만 비우기 (구조 유지 + id 리셋) 파일은 두고 행만 삭제. id를 1부터 다시 시작하려면 sqlite_sequence도 비움. 
sqlite3 sensor_data.db "DELETE FROM readings; DELETE FROM sqlite_sequence WHERE name='readings'; VACUUM;"
VACUUM은 삭제 후 파일 크기를 실제로 줄입니다.
방법 ③ 백업 후 초기화 (데이터 보존) 기존 데이터가 아까우면 백업부터. 
cp sensor_data.db ~/sensor_backup_$(date +%F_%H%M).db    # 먼저 백업
rm -f sensor_data.db sensor_data.db-wal sensor_data.db-shm   # 그다음 초기화
💡 권장테스트 데이터를 버릴 땐 방법 ①(통째 삭제, 가장 깔끔). 실수집 데이터가 섞여 아까우면 방법 ③(백업 후 삭제). 어느 경우든 수집 프로세스를 먼저 멈추고 실행하세요(돌면서 지우면 충돌).

━━ (다) 프로젝트 통째 정리 ━━

완전히 끝낼 때. 순서가 중요(서비스 먼저, 폴더 나중).

① systemd를 등록한 경우 — 먼저 내리기

⚠️ 왜 먼저인가서비스가 등록된 채 폴더를 지우면, systemd가 사라진 파일을 계속 실행 시도(Restart=always)하며 로그가 폭발합니다. 반드시 서비스부터 정리하세요.
# 자동시작 해제 + 지금 중지 (둘 다)
sudo systemctl disable --now multinode_sensor_demo_hub multinode_sensor_demo_dashboard

# 서비스 파일 삭제
sudo rm /etc/systemd/system/multinode_sensor_demo_hub.service
sudo rm /etc/systemd/system/multinode_sensor_demo_dashboard.service

# systemd에 반영 + 실패기록 정리
sudo systemctl daemon-reload
sudo systemctl reset-failed

App Lab만 썼다면 이 단계는 건너뜁니다(App Lab에서 앱 삭제로 충분).

노드를 다른 용도로 쓰거나 WiFi 정보를 지우려면 — Arduino IDE에서 빈 스케치(File → Examples → 01.Basics → BareMinimum)를 업로드하면 펌웨어·WiFi 설정이 모두 지워집니다.

⑥ uv는 보통 유지

uv 본체(~/.local/bin/uv)와 캐시(~/.cache/uv)는 폴더 밖에 있어 영향받지 않습니다. 다음 프로젝트에서 또 쓰므로 두는 편이 좋습니다.

✅ 정리 체크리스트완전 종료 시: ① 서비스 disable+삭제 → ② (백업) → ③ 폴더 rm -rf → ④ (SSH 끄기) → ⑤ (노드 BareMinimum). 이렇게 하면 새 프로젝트와 완전히 분리됩니다. venv는 폴더 안에 격리되고, systemd 흔적은 파일삭제+daemon-reload로 사라집니다.
